# Ez.Bokasafn — Design Spec

**Date:** 2026-04-25
**Status:** Approved (pending user spec review)

## 1. Goal

A mobile-first PWA that answers one question better than `leitir.is`: **"Is this book on the shelf right now, and which branch do I go to?"**

The audience is casual Icelandic readers, not researchers. Search results today bury availability under noise (e-books, journals, theses, articles); we surface availability and strip the rest.

## 2. Scope

### In scope
- Physical books only.
- All Icelandic public libraries in the consortium (`354ILC_NETWORK`).
- Pure discovery: search, see availability, see where to go.
- UI in Icelandic only.
- Mobile-first PWA. Desktop renders correctly but is not the design target.
- Single deployment with library as a runtime filter.

### Out of scope
- E-books, audiobooks, films, articles, journals, theses, sheet music, any non-physical-book content.
- User accounts, holds/reservations, loan history, renewals.
- Bilingual UI.
- Recommendations, related items, "browse by author" surfaces.
- Service Worker / offline shell (future).
- Rate limiting (future — origin check only for v1).

## 3. Architecture

```
                       ┌──────────────────────────────────┐
   Phone browser  ───▶ │  Azure Static Web Apps           │
                       │  ┌────────────────┐              │
                       │  │ Svelte SPA     │  static      │
                       │  │ (3 screens)    │  /index.html │
                       │  └───────┬────────┘              │
                       │          │ fetch /api/...        │
                       │  ┌───────▼─────────┐             │
                       │  │ Azure Functions │             │
                       │  │ (TypeScript)    │             │
                       │  │  - /suggest     │  proxy +    │
                       │  │  - /search      │  jwt cache  │
                       │  │  - /book/:id    │             │
                       │  └───────┬─────────┘             │
                       └──────────┼──────────────────────-┘
                                  ▼
                          leitir.is (Primo / Alma)
```

**Why this shape**

- The Function does three jobs the browser cannot:
  1. Hold the leitir.is guest JWT (the JWT is IP-bound; browsers leak the user's IP, the Function gives one stable origin).
  2. Bypass CORS (the SPA cannot call `leitir.is` directly from another origin).
  3. Cache the JWT across all anonymous callers (one shared token, refreshed before expiry).
- The SPA owns all UX state. The URL is the source of truth for shareable state (`?q=...&lib=...`). No server session, no cookies.
- One Azure Static Web App resource. The Function is "linked" — same hostname, same deployment, atomic releases.

**Stack**
- Frontend: Svelte + Vite (SPA), TypeScript.
- Backend: Azure Functions on Consumption plan, TypeScript (Node 20).
- Hosting: Azure Static Web Apps (free tier).
- CI/CD: GitHub Actions (auto-generated by Azure SWA).

## 4. Frontend

### Screens

| Route | File | Purpose |
|---|---|---|
| `/` | `routes/+page.svelte` | Home — search input + library scope picker |
| `/search` | `routes/search/+page.svelte` | Results grouped by availability |
| `/book/:mmsId` | `routes/book/[mmsId]/+page.svelte` | Detail — book-first hero + branch availability |

URL is the canonical state. `/search?q=hringadrottinssaga&lib=10000_BORG` is shareable and bookmarkable.

### Home

Layout from top to bottom:

1. **Library scope picker** in the top-left, anchored above the wordmark. Pill style, leading dot in the warm-rust `--accent` color, caret on the right. Tap → bottom sheet listing all consortium libraries; selection persists to `LocalStorage`.
2. **Editorial wordmark** `Finna bók` set in italic Newsreader, ~88 px, with a 6 px moss-green underline under the word `bók`. Two-line stack: `Finna` left-aligned, `bók` indented 24 px on the second line.
3. **Tagline** in italic Newsreader, ~15 px, max-width 240 px: `Á hvaða safni er hún til, og er hún á hillunni núna?`
4. **Search field** — placeholder `Leita að bók, höfundi…`. 48 px tall, 14 px radius, leading magnifier icon, `--bg-elev` background.

The picker tap opens the scope sheet; everything else stays focused on getting the user to a query.

(The design prototype shows a "Nýlega leitað" recent-searches block under the search field — deferred from v1; see §11.)

### Results

Top header: search input + library scope picker (compact form).

Results body grouped into two sections:

1. **`Laus (n)`** — books with at least one available copy in the chosen scope. Green section header.
2. **`Í útláni (n)`** — books that exist in the catalogue but have no available copy. Muted section header.

Each row:
- Cover thumbnail (38×56) — see "Cover thumbnails" below.
- Title (bold).
- `Höfundur · Ár` (secondary).
- Third line: branches where the book is on shelf (`Grófin · Sólheimar · Kringlan`) for available items; `Skilað í fyrsta lagi: 8. maí` for on-loan items. Truncate to 3 branches + `+n` overflow.

Empty results: `Engin niðurstaða fyrir „<q>"`. If the upstream `did_u_mean` field is set, render it as a tap-to-search prompt below the message.

Pagination: "Show more" button at the bottom; passes `&offset=` to `/api/search`.

### Detail

Book-first layout:

- **Floating app bar** over the hero — back button (left) + share button (right). Each is a 36 px circular icon button with `backdrop-filter: blur(12px)` over a translucent `--bg-elev`. Share uses the Web Share API where available; falls back to copying the canonical URL to clipboard with a toast.
- **Hero** — radial-gradient tint derived from the cover's dominant color, fading to `--bg` near the bottom; centered large cover (168×252), title in Newsreader 28 px, italic author byline, facts row (`1934 · 624 bls. · Skáldsaga`).
- **Tags** — pill-shaped chips below the hero, centered, listing genres/format from the PNX record (e.g. `Skáldverk` `Íslensk` `Sveitalíf` `Klassík`).
- **Summary** — short Icelandic description in Newsreader 15 px, line-height 1.55.
- **Branch availability table** — single bordered card. Header: `Staða eftir safni` with count pill (`6 söfn · 4 á hillunni`). Each row: branch name (Inter 15 px) + call number (JetBrains Mono 11.5 px) on the left; status on the right (`Á hillu` with leading moss dot for available; `Í útláni` plus due date `Skilað 8. maí` for on loan).

If the record is not found upstream: render `Bókin fannst ekki` with a back link to the previous results.

### Shared components

| Component | Responsibility |
|---|---|
| `<SearchHeader>` | Input + autocomplete dropdown + library scope picker. 3+ chars → 200ms debounce → `/api/suggest`. AbortController cancels stale requests. |
| `<ResultRow>` | One row in the results list. |
| `<AvailabilitySection>` | Section header (`Laus (n)` / `Í útláni (n)`) + list of `<ResultRow>`. |

### Cover thumbnails

`<BookCover>` tries sources in order, falling through silently:
1. `https://baekur.is/cover/tbn/<id>` (302 → image).
2. ExLibris/Syndetics proxy by ISBN: `https://proxy-euf.hosted.exlibrisgroup.com/exl_rewrite/syndetics.com/index.php?client=primo&isbn=<isbn>/sc.jpg`.
3. Grey placeholder with the title's initials.

Lazy-loaded with `loading="lazy"`. Loading errors do not surface to the user.

## 5. Backend

### Endpoints

All endpoints live under `/api/*` and call `assertAllowedOrigin()` as their first line.

| Method | Path | Inputs | Output |
|---|---|---|---|
| GET | `/api/suggest` | `q`, `lib` | `{ suggestions: string[] }` |
| GET | `/api/search` | `q`, `lib`, `offset?` | `{ available: Book[], onLoan: Book[], total: number, didYouMean?: string }` |
| GET | `/api/book/:mmsId` | `lib` | `{ book: BookDetail, branches: BranchAvailability[] }` |

### Modules

| Module | Responsibility |
|---|---|
| `lib/jwtCache.ts` | Module-scoped guest JWT. `getJwt()` returns the cached token; refreshes if missing or within 30 minutes of `exp`. Refresh is mutex-guarded so a burst of requests on cold start makes one upstream call, not many. |
| `lib/leitir.ts` | Typed thin wrapper around the upstream calls in `leitir-api-notes.md`. Returns typed responses, throws typed errors. |
| `lib/shape.ts` | Turns verbose Primo `pnxs` / `delivery` / `getPhysicalService` payloads into the trimmed shapes the SPA consumes. **This is where the "less clutter" promise is implemented.** Filters out non-physical-book records, buckets results by availability, drops fields the UI never uses. |
| `lib/originCheck.ts` | `assertAllowedOrigin(req)` — first line of every handler. Allow-list against `ALLOWED_ORIGINS`. |
| `lib/errors.ts` | `HttpError` class with status + safe message. Top-level handler maps unknown errors to 502. |

### Upstream call patterns

- **`/api/suggest`** — single upstream call to `/primaws/rest/pub/suggest`.
- **`/api/search`** — parallel fan-out: `/primaws/rest/pub/pnxs` + `POST /primaws/rest/pub/delivery`. `shape.ts` merges by `mmsId`.
- **`/api/book/:mmsId`** — parallel fan-out: `/primaws/rest/priv/nz/pnx/P/<mmsId>` (full record) + `/primaws/rest/pub/getPhysicalService/<mmsId>` (per-branch holdings). Optionally a follow-up to `/primaws/rest/priv/ILSServices/titleServices/<svcId>` if shelf detail is missing.

### JWT refresh

- Cold start: lazy — first request triggers `fetchGuestJwt()`.
- Warm: served from the module-level `let cachedJwt`.
- Within 30 minutes of expiry: refresh on next call (do not block on it; serve the still-valid current token while a background refresh updates `cachedJwt`).
- Upstream returns 401/403 on a request: refresh once, retry once. Persistent failure → 502 to the client.
- Each Function instance has its own cache. Acceptable cost: each new instance pays one extra `guestJwt` call. If this becomes meaningful, escalate to Azure Table Storage (deferred).

### Shared types

`packages/types/index.ts` — referenced by both `apps/web` and `apps/api` via TS path alias. No build step.

```ts
export interface Book {

  title: string;
  author?: string;
  year?: number;
  isbn?: string;
  coverSources: string[]; // ordered fallback chain
  branchesOnShelf: string[]; // names only, for the row third-line
  earliestReturn?: string; // ISO date, only for onLoan
}

export interface BookDetail extends Book {
  summary?: string;
  genres: string[];
  pageCount?: number;
}

export interface BranchAvailability {
  branch: string;
  status: 'on-shelf' | 'on-loan';
  callNumber?: string;

}

export interface LibraryScope {
  code: string;   // e.g. "10000_BORG"
  label: string;  // e.g. "Borgarbókasafn"
}
```

## 6. Data flow

### App boot

1. SPA loads. Reads `?lib=` from URL, falls back to `LocalStorage.lastScope`, then `VITE_DEFAULT_SCOPE` seed if neither is set.
2. Function: cold start triggers no work; first `/api/*` call initializes the JWT.

### Type-to-search

```
User types in <SearchHeader>
  ↓ (debounce 200ms; fire at >= 3 chars)
GET /api/suggest?q=&lib=
  ↓ Function: assertAllowedOrigin → getJwt → upstream /suggest
  ↓ shape: keep only `text` strings, dedupe
Dropdown updates. AbortController cancels stale requests.
```

### Submit search

```
SearchHeader.submit(q)
  ↓ navigate to /search?q=&lib=
Results page mounts → GET /api/search?q=&lib=&offset=0
  ↓ Function fan-out (parallel):
     - /primaws/rest/pub/pnxs
     - POST /primaws/rest/pub/delivery
  ↓ shape.ts:
     - filter to physical books only
     - bucket each into available / onLoan based on delivery data
     - keep only fields used by the UI
  ↓ Response: { available, onLoan, total, didYouMean? }
SPA renders two <AvailabilitySection>s.
```

### View detail

```
Tap row → navigate to /book/:mmsId?lib=
Detail page mounts → GET /api/book/:mmsId?lib=
  ↓ Function fan-out (parallel):
     - /primaws/rest/priv/nz/pnx/P/<mmsId>
     - /primaws/rest/pub/getPhysicalService/<mmsId>
     - (optional) /primaws/rest/priv/ILSServices/titleServices/<svcId>
  ↓ shape.ts → { book, branches }
SPA renders hero + summary + <BranchTable>.
```

### Library scope change

User opens `<LibraryScopePicker>` → selects → URL is rewritten with new `?lib=` → Svelte router triggers refetch automatically. Selection is persisted to `LocalStorage.lastScope`.

### Caching

- Function-side: only the JWT.
- Browser-side: Svelte route stores keep the last results in memory so back-navigation from detail → results is instant.
- No HTTP caching headers on `/api/*` responses for v1 (data is fresh; load is small).

### Origin check

```ts
// lib/originCheck.ts
export function assertAllowedOrigin(req: HttpRequest): void {
  const allowed = (process.env.ALLOWED_ORIGINS ?? '')
    .split(',').map(s => s.trim()).filter(Boolean);
  const origin = req.headers.get('origin') ?? req.headers.get('referer') ?? '';
  if (!allowed.some(a => origin.startsWith(a))) {
    throw new HttpError(403, 'Forbidden');
  }
}
```

First line of every `/api/*` handler. Allow-list, not deny-list.

## 8. Observability & analytics

### Server-side: Application Insights

Enable Application Insights on the Function App (built-in integration; free tier covers 5 GB/month — well above this app's needs).

Automatic instrumentation captures:
- Every `/api/*` request: status, latency, caller IP (anonymized).
- Exceptions and stack traces.
- Outbound dependencies: each `leitir.is` call appears as a dependency with its own latency chart.

Custom events emitted from handlers (use `context.log` with structured fields):

| Event | Fields | Why |
|---|---|---|
| `search_performed` | `lib`, `total`, `qLength` | What scopes are used; how often searches return zero |
| `book_detail_viewed` | `mmsId`, `lib` | Which records get clicked through |
| `library_scope_changed` | `fromLib`, `toLib` | How users move between libraries |
| `jwt_refreshed` | `reason: 'expiry'` \| `'auth-error'` | Health of token cache |
| `cover_fallback` | `source: 'baekur'` \| `'syndetics'` \| `'placeholder'` | Tells us which cover sources actually deliver |

**Privacy note:** Log query *length* and *result count*, not the raw query string — search terms can be PII when joined with IP. Log scope codes and `mmsId`s freely (non-personal).

**Alerts:** 5xx rate > 5% over 5 min; any JWT refresh failure in a 1h window.

### Client-side: Cloudflare Web Analytics

Free, cookieless, GDPR-friendly. Captures page views, unique visitors (no fingerprinting), referrers, country, browser, OS.

- Add the Cloudflare Web Analytics `<script>` tag to `index.html` only in production builds (gate via `import.meta.env.PROD`).
- No custom events client-side; product-level events live in App Insights.
- No cookie banner needed (Cloudflare uses no cookies and no persistent identifiers).

## 9. Error handling & edge cases

### Frontend states

| Surface | Empty | Loading | Error |
|---|---|---|---|
| Autocomplete | (no dropdown) | (no spinner — too jumpy at small sizes) | Silent — fall back to plain input |
| Results | `Engin niðurstaða fyrir „<q>"` + tap-to-try `<didYouMean>` if present | Skeleton rows × 4–6 | `Eitthvað fór úrskeiðis. Reyndu aftur.` + Retry button |
| Detail | n/a (404 path) | Skeleton hero + table | `Bókin fannst ekki` + back link |
| Cover | Initials placeholder | Progressive image load | Silent fall-through chain |

### Backend error mapping

| Upstream condition | Function returns | SPA shows |
|---|---|---|
| 401/403 from leitir.is | Refresh JWT once, retry once. Persistent failure → 502 | Generic error |
| 404 (record gone) | 404 | `Bókin fannst ekki` |
| Network timeout (>10s) | 504 | Generic error + retry |
| Origin not allowed | 403 | (won't be reached from the SPA) |
| Anything else | 502 | Generic error + retry |

JWT refresh is silent — the SPA never sees it.

### Edge cases

- **`did_u_mean`** — when results are empty but the field is set, render the suggestion as a tap-to-search prompt; do not auto-redirect.
- **Diacritic normalization** — pass user input through untouched. The upstream `/suggest` already returns both accented and unaccented forms.
- **Long branch lists** on results rows — truncate to 3 branches + `+n`. Full list visible on detail.
- **Direct deep-link to `/book/:mmsId` with no scope** — fall back to consortium-wide search; show a hint `Engin staðbundin sía valin.`
- **Stale results in browser back-cache** — accept it. Availability that's a few minutes old is fine for v1.
- **Mobile data / 3G** — small responses + skeleton placeholders carry the experience.

## 11. Deferred (post-v1)

- Rate limiting per IP (in-memory or Azure Table Storage).
